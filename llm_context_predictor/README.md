# llm_context_predictor

This repository consist of `Context predictor` of the proposed interface. The context predictor is implemented with the help of `Llama-index` 
python package. The file organization is as follows,

```
├── cfg
│   └── vlm_params.yaml    ---> File consisting of parameters to initialize the VLMs
├── system_prompt          
│   └── system_prompt.txt  ---> The system prompt used by the VLM to extract context
├── context_predictor.py   ---> The context predictor is implemented in this file as VLMContextPredictor()
├── __init__.py 
├── mm_init.py             ---> In this file initialisation of various models were done
├── test.ipynb
└── utils.py               ---> Helper functions to clean the JSON string generated by the VLM
```

## Installation
This package can be installed by the following command
```
pip install -e .
```

## Configuration file

```
api: # The API to use eg: openai, mistral, gemini, ollama (for local llms)
model: #[str] The VLM model to use.
# temperature: #[float] The temperature to use for sampling (default :  DEFAULT_TEMPERATURE=0.1)
max_tokens: #[int] The maximum numbers of tokens to generate, ignoring the number of tokens in the prompt (default : 300)
context_window: #[int] The maximum number of context tokens for the model. (default :  DEFAULT_CONTEXT_WINDOW=3900)
image_detail: #[str] The level of details for image in API calls. Can be low, high, or auto.
max_retries: #[int] Maximum number of retries (default=3)
timeout: #[float] The timeout, in seconds, for API requests. (default=60.0s)
api_key: #[str] The API key. (can be set as env parameter)
# api_base: #[str] The base URL for API.
# api_version: #[str] The API version for API.
```

## Usage

```
from llm_context_predictor.context_predictor import VLMContextPredictor

context_predictor = VLMContextPredictor(cfg_file_path="<path to the cfg file if needs to be changed>")
context_predictor.predict_context("Image path") #Example: test.png

```
